This project features a full-stack Twitter clone that allows users to create profiles, make tweets, and like otherâ€™s tweets which are stored in a MongoDB database and displayed in a react application. The database stores two kinds of Schema: Users and Tweets. The User schema stores a username, password, and a list of liked tweets. The Tweet Schema stores the body text of the tweet itself as well as the ID of the user who created it and a timestamp. When a user does an action, such as creating a tweet or account, The frontend React application makes an Axios REST request to the Express backend which has many routes for both users and tweets to handle the requests. When the frontend gets the data it needs back it dynamically updates the front end. Many front end elements like the lists of tweet cells are resued to demonstrate the power of using a framework like React.
